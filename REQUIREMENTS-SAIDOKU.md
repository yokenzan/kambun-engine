## 再読文字（角括弧記法対応）要求仕様 v0.1

### 目的
- **目的**: 再読文字に対し、第二読（前置）と第一読（後置）を所定位置に出力する。
- **角括弧記法**: `<…>` により第一読の語形（例: ス, ズ）を入力上で明示できるようにする。
- **両立**: 既存のレ点・一二点などの訓点による語順決定と両立する。

### 用語
- **再読文字**: `*` を前置してマークする文字（例: `*将`）。
- **第二読**: 再読文字の前置的な読み（例: 将→「まさに」, 未→「いまだ」）。
- **第一読**: 再読文字の後置的な読み（例: 将→「す」, 未→「ず」）。
- **角括弧記法**: `*将(まさ)ニ<ス>` の `<ス>` 部分。第一読の語形を指定。

### 入力記法（Phesoca拡張）
- **再読マーク**: `*` 直後の1文字（単字のみ対象）を再読文字として扱う。
- **第二読の指定**: 再読文字に付与された振り仮名 `(よみ)` を第二読の基底読みとする。再読文字に続くカタカナ送り仮名は第二読に連結し、出力時に平仮名化する。
  - 例: `*将(まさ)ニ` → 第二読は「まさ」+「に」=「まさに」。
- **第一読の指定（角括弧記法）**: 再読文字の直後に書かれた `<…>` の内容を第一読として採用。内容は仮名（ひらがな/カタカナ）を受け付け、出力時に平仮名へ正規化する。
  - 例: `<ス>` → 「す」, `<ズ>` → 「ず」。
- **既定辞書（フォールバック）**:
  - `<…>` がない場合の第一読・第二読は既定辞書で補う。
  - 最低限の要件:
    - 将: 第二読「まさに」、第一読「す」
    - 未: 第二読「いまだ」、第一読「ず」
  - 今回の必須テストは将のみ。
- **他記法との関係**:
  - 訓点 `[レ]` や `[一二]` は従来通り読み順の依存関係を与える。
  - 置き字 `_` は出力しない。
  - 助字 `~` は従来通り振り仮名で読む（再読処理とは独立）。

### 読み順・配置ルール
- **基本方針**: 読み順は依存グラフ（訓点）で決定するが、再読文字は1箇所から2つの出力トークン（第二読・第一読）に分解し、各々をスコープの先頭・末尾に配置する。
- **スコープ**:
  - 再読文字 S から見て「S を後に回す」依存（例: S に付いた `[レ]` による「次語→S」）の対象語群をスコープとする。
  - 実装簡便化の最小要件として、トポロジカル順で S より前に読まれる連続区間（句読点・文末で打ち切り）をスコープとみなしてよい。
- **配置**:
  - 第二読はスコープの先頭出力直前に挿入する（スコープのヘッドの直前）。
  - 第一読はスコープの末尾直後に挿入する。句読点が直後に続く場合は句読点の直前に置く。
  - スコープが判定できない（依存が無い）場合は、直後の語を1語スコープとし、句読点までを伸張してよい。
- **句読点の扱い**:
  - `、` `。` `，` `．` `；` `：` はスコープ境界。第一読は句読点の直前へ。

### 生成ルール
- **第二読生成**:
  - 優先順位: 振り仮名 `(よみ)` → 既定辞書。
  - 送り仮名（カタカナ）は第二読に連結し、出力時は平仮名化。
  - 置き字は無視。
- **第一読生成**:
  - 優先順位: 角括弧 `<…>` → 既定辞書。
  - 出力時は平仮名化。送り仮名は付与しない（角括弧内に含める場合のみその文字列を用いる）。
- **再読でない語**:
  - 助字/助動詞は振り仮名読み、その他は漢字+送り仮名（平仮名化）。

### 互換性
- **非再読ケース**: 既存の非再読テストは挙動を変更しない。
- **角括弧の適用範囲**: `<…>` は再読文字の直後にのみ有効。その他の位置で現れた場合は無視（最小要件）。

### 受入要件（検証観点）
- **パーサ**:
  - `*` で再読フラグを立てられる。
  - 再読文字直後の `<…>` を検出し、現在の語の `saidokuReading`（第一読）に格納する。
  - `<…>` 内のカタカナ/ひらがなを受け付け、出力時に平仮名へ正規化される。
  - `<…>` は `kanji` や `okurigana` へ混入しない。
- **読み順展開**:
  - 再読文字は `ReadingUnit` 2件（`SAIDOKU_SECOND` と `SAIDOKU_FIRST`）に展開される。
  - 配置は「スコープ先頭の直前に第二読」「スコープ末尾の直後に第一読」（句読点があればその直前）。
  - 訓点により S 自身が後置（例: `[レ]`）されていても、第二読はスコープ先頭へ前置される。
- **文字列生成**:
  - 第二読は `(ふりがな)+送り仮名` を平仮名で出力。
  - 第一読は `<…>`（または辞書）を平仮名で出力。
  - 句読点の直前に第一読が来る。

### テストでの受入基準
- **入力**: `*将(まさ)ニ<ス>[レ]来タラムト。`
- **期待出力**: `将に来たらむとす。`
  - 「将に」= 第二読（「まさ」+「ニ」）の平仮名化
  - 「来たらむと」= レ点によるスコープ本体
  - 「す」= 第一読（`<ス>`）
  - 「。」の直前に「す」が配置

### 追加参考（任意・推奨）
- **例1**: `*未(いま)ダ<ズ>[レ]知ラズ。` → `いまだ知らず。`
- **例2（レ点なし）**: `*将(まさ)ニ<ス>行カント。` → `将に行かんとす。`
  - スコープは直後から句読点まで。第二読は先頭、第一読は末尾。

### エラーハンドリング（最小）
- **無効な角括弧**: `<…>` が再読文字直後以外に現れた場合は無視。
- **空の角括弧**: `<…>` が空/記号のみは無視し、辞書にフォールバック。

### 非目標（今回扱わない）
- **複合再読**: 再読文字の複合文字対応（単字のみ対象）。
- **連鎖最適化**: 再読文字が複数連鎖する高度な最適配置。
- **多義詳細**: 可・当・宜などの詳細活用一式。
