# 読み順解決アルゴリズムの解説（中学生向け）

## そもそも何が問題なの？

漢文を日本語で読むとき、**書かれている順番と、読む順番が違う**んです。

### 例えば：料理のレシピで考えてみよう

料理のレシピには順番がありますよね：

```
① 野菜を切る
② フライパンを温める
③ 油を入れる
④ 野菜を炒める
```

これは上から順番に読めばOKです。

でも、もしこんなレシピがあったら？

```
野菜を炒める　←　油を入れる　←　フライパンを温める　←　野菜を切る
```

矢印（←）がついていて、「右から順番に読んでね」という指示があります。
これが漢文の返り点と同じ仕組みです！

## 漢文の問題：バラバラの順番を整理する

漢文にはこんな印（訓点）がついています：

- **レ点**：「次を先に読んでから戻る」という印
- **一二点**：「①の位置にジャンプして読む」という印

例：
```
愛[レ]人　→　「人」を先に読んで、「愛」に戻る
　　　　　　　読む順番：人 → 愛
```

**私たちの課題**：訓点を見て、正しい読む順番を見つけること！

## 解決方法：「やることリスト」を作る

### ステップ1：料理で考えてみよう

カレーを作るとき：

```
- お米を炊く（30分）
- 野菜を切る（10分）
- 肉を炒める（5分）　← 野菜を切ってからじゃないとできない
- カレールーを入れる（20分）　← 肉を炒めてからじゃないとできない
```

「やることリスト」はこうなります：

```
野菜を切る → 肉を炒める → カレールーを入れる

同時進行できる：お米を炊く
```

これを**依存関係**といいます。「AをしないとBができない」という関係のことです。

### ステップ2：漢文でも同じことをする

```
不[レ]覚[レ]暁

訓点のルール：
- 覚[レ] → 「暁」を先に読んでから「覚」を読む
- 不[レ] → 「覚」を先に読んでから「不」を読む

やることリスト：
暁 → 覚 → 不

読む順番が決まった！
```

## 具体的な仕組み：矢印でつなぐ

### 簡単な例：愛[レ]人

```
文字：  愛[レ]  人
番号：   0      1

訓点のルール「レ点」：次を先に読む

矢印で表すと：
1(人) → 0(愛)

読む順番：1 → 0
つまり：人 → 愛
```

### もう少し複雑な例：不[レ]覚[レ]暁

```
文字：  不[レ]  覚[レ]  暁
番号：   0      1      2

訓点のルール「レ点」を2回使う：
- 覚[レ]：2(暁) → 1(覚)
- 不[レ]：1(覚) → 0(不)

矢印でつなぐと：
2(暁) → 1(覚) → 0(不)

読む順番：2 → 1 → 0
つまり：暁 → 覚 → 不
```

こんな風に、矢印でつながった順番を見つけるのが目標です。

## アルゴリズムの本体：どうやって順番を決める？

### 「できることから始める」作戦

カレーの例に戻ります：

```
やることリスト：
- お米を炊く（他に何も必要ない）
- 野菜を切る（他に何も必要ない）
- 肉を炒める（野菜を切ってから）
- カレールーを入れる（肉を炒めてから）
```

**ステップ1**：「他に何も必要ないもの」を探す
→ お米を炊く、野菜を切る

**ステップ2**：それを実行する
→ ✓ お米を炊く
→ ✓ 野菜を切る

**ステップ3**：できるようになったことを探す
→ 肉を炒める（野菜を切ったから、できる！）

**ステップ4**：それを実行する
→ ✓ 肉を炒める

**ステップ5**：できるようになったことを探す
→ カレールーを入れる（肉を炒めたから、できる！）

**ステップ6**：それを実行する
→ ✓ カレールーを入れる

**完成！**

### これが「トポロジカルソート」です

難しい名前ですが、やっていることは単純：

1. 「待たなくていいもの」を見つける
2. それを実行リストに追加
3. 実行したものを消す
4. 1に戻る

これを繰り返すだけです。

## 実際の漢文で試してみよう：愛[レ]人

### 準備：矢印を作る

```
文字：  愛[レ]  人
番号：   0      1

矢印：1(人) → 0(愛)
```

### 「待ち数」を数える

それぞれの文字について、「何個の矢印が入ってくるか」を数えます。

```
0(愛)：矢印が1本入ってくる（人から）→ 待ち数 = 1
1(人)：矢印が入ってこない　　　　　　　→ 待ち数 = 0
```

待ち数 = 0 は「すぐに読める」という意味です。

### 順番を決める

**現在の状況**
```
待ち数：
0(愛) = 1  ← まだ待つ
1(人) = 0  ← すぐ読める！

読める文字リスト：[1(人)]
完成リスト：[]
```

**ステップ1**：人を読む
```
1(人)を読んだ！

1(人) → 0(愛) の矢印を消す
→ 0(愛)の待ち数が 1 → 0 になった！

待ち数：
0(愛) = 0  ← 読める！

読める文字リスト：[0(愛)]
完成リスト：[1(人)]
```

**ステップ2**：愛を読む
```
0(愛)を読んだ！

読める文字リスト：[]
完成リスト：[1(人), 0(愛)]
```

**完成！読む順番：1 → 0（人 → 愛）**

## もっと複雑な例：施[二]於人[一]

一二点は「ジャンプ」を意味します。

```
文字：  施[二]  於  人[一]
番号：   0      1   2

訓点のルール：
- [一]が起点（スタート）
- [二]が終点（ゴール）
- 起点 → 間の文字 → 終点 の順で読む

読む順番：2(人) → 1(於) → 0(施)
```

### 矢印を作る

```
2(人) → 1(於)  ← 起点の次の文字へ
1(於) → 0(施)  ← 終点へ
```

### 待ち数を数える

```
0(施)：矢印が1本入ってくる（於から）→ 待ち数 = 1
1(於)：矢印が1本入ってくる（人から）→ 待ち数 = 1
2(人)：矢印が入ってこない　　　　　　　→ 待ち数 = 0
```

### 順番を決める

**初期状態**
```
待ち数：
0(施) = 1
1(於) = 1
2(人) = 0  ← すぐ読める！

読める文字リスト：[2(人)]
完成リスト：[]
```

**ステップ1**：人を読む
```
2(人) → 1(於) の矢印を消す
→ 1(於)の待ち数が 1 → 0

待ち数：
0(施) = 1
1(於) = 0  ← 読める！

読める文字リスト：[1(於)]
完成リスト：[2(人)]
```

**ステップ2**：於を読む
```
1(於) → 0(施) の矢印を消す
→ 0(施)の待ち数が 1 → 0

待ち数：
0(施) = 0  ← 読める！

読める文字リスト：[0(施)]
完成リスト：[2(人), 1(於)]
```

**ステップ3**：施を読む
```
読める文字リスト：[]
完成リスト：[2(人), 1(於), 0(施)]
```

**完成！読む順番：2 → 1 → 0（人 → 於 → 施）**

## まとめ：アルゴリズムの流れ

### 1. 矢印を作る（依存関係グラフ）

訓点のルールから、「AよりBを先に読む」という矢印を作る

```
例：愛[レ]人
→ 人 → 愛 という矢印
```

### 2. 待ち数を数える

それぞれの文字に「何本の矢印が入ってくるか」を数える

```
人：0本（すぐ読める）
愛：1本（人を待つ）
```

### 3. 「待ち数0」から始める

待ち数が0の文字は、すぐに読めます

```
人を読む
```

### 4. 読んだら矢印を消す

読んだ文字から出ている矢印を消して、相手の待ち数を減らす

```
人 → 愛 の矢印を消す
→ 愛の待ち数が 1 → 0
```

### 5. 繰り返す

また「待ち数0」を探して読む

```
愛を読む
```

### 6. 全部読んだら完成！

```
読む順番：人 → 愛
```

## なぜこの方法が良いの？

### 1. 複雑でも大丈夫

訓点がどれだけ複雑でも、「矢印を作る」→「順番に処理する」だけでOK

### 2. 間違えない

ルール通りに機械的に処理するので、間違えません

### 3. 追加しやすい

新しい訓点（上中下点など）を追加するときも、「矢印の作り方」を増やすだけ

### 4. 理解しやすい

「待ち数」という分かりやすい概念だけで説明できる

## 用語解説

### 依存関係グラフ

「矢印でつないだ図」のこと。「AをBより先に」という関係を矢印で表します。

### トポロジカルソート

「矢印の順番を守りながら、全部並べる」こと。このプログラムでは「待ち数0から順に処理する」方法を使っています。

### 待ち数（入次数）

「その文字を読む前に、何個の文字を読まないといけないか」の数。
待ち数が0なら、今すぐ読めます。

### Kahn's アルゴリズム

「待ち数0から順に処理していく」方法の名前。人の名前（Kahn さん）がついているだけで、難しいことではありません。

## もっと例を見てみよう

### 例1：レ点が3つ連続（難易度：★★☆）

```
不[レ]覚[レ]暁[レ]春
```

**訓点のルール：**
- 暁[レ]：春 → 暁
- 覚[レ]：暁 → 覚
- 不[レ]：覚 → 不

**矢印を作る：**
```
3(春) → 2(暁) → 1(覚) → 0(不)
```

**待ち数：**
```
0(不) = 1  ← 覚を待つ
1(覚) = 1  ← 暁を待つ
2(暁) = 1  ← 春を待つ
3(春) = 0  ← すぐ読める！
```

**順番を決める：**
1. 春を読む（待ち数0）→ 暁の待ち数が0になる
2. 暁を読む（待ち数0）→ 覚の待ち数が0になる
3. 覚を読む（待ち数0）→ 不の待ち数が0になる
4. 不を読む（待ち数0）

**結果：春 → 暁 → 覚 → 不**

### 例2：一二三点（難易度：★★★）

```
読[三]書[二]於家[一]
```

**訓点のルール：**
- [一]が起点（スタート地点）
- [二]と[三]が終点（ゴール地点）
- 起点 → 一番近い終点 → 次の終点 の順

**読む順番：**
```
3(家[一]) → 2(於) → 1(書[二]) → 0(読[三])

つまり：家 → 於 → 書 → 読
```

**矢印を作る：**
```
3(家) → 2(於)  ← 起点の次へ
2(於) → 1(書)  ← 最初の終点へ
1(書) → 0(読)  ← 次の終点へ
```

**待ち数：**
```
0(読) = 1  ← 書を待つ
1(書) = 1  ← 於を待つ
2(於) = 1  ← 家を待つ
3(家) = 0  ← すぐ読める！
```

**処理の流れ：**
1. 家を読む → 於の待ち数が0に
2. 於を読む → 書の待ち数が0に
3. 書を読む → 読の待ち数が0に
4. 読を読む

**結果：家 → 於 → 書 → 読**

### 例3：レ点と一二点の混合（難易度：★★★★）

```
愛[レ]読[二]書[一]
```

**訓点のルール：**
- 読[二]書[一]：書 → 読（一二点のルール）
- 愛[レ]：次を先に読む（レ点のルール）

**矢印を作る：**
```
2(書) → 1(読)  ← 一二点の効果
1(読) → 0(愛)  ← レ点の効果
```

**待ち数：**
```
0(愛) = 1  ← 読を待つ
1(読) = 1  ← 書を待つ
```

**処理の流れ：**
1. 書を読む → 読の待ち数が0に
2. 読を読む → 愛の待ち数が0に
3. 愛を読む

**結果：書 → 読 → 愛**
**意味：書を読むことを愛す**

### 例4：実際の有名な漢文（難易度：★★★★★）

```
己[二]所[レ]不[レ]欲[一]
```

これは「己所不欲、勿施於人」（己の欲せざる所、人に施す勿れ）の一部です。

**訓点のルール：**
1. 欲[一]が起点、所[二]が実質的な終点
2. 不[レ]：欲 → 不
3. 所[レ]：不 → 所（実際には、レ点が連続する場合）

実装上の矢印：
```
3(欲) → 2(不)  ← 不[レ]の効果
2(不) → 1(所)  ← 所[レ]の効果
3(欲) → 1(所)  ← 一二点の効果（起点→終点）
```

**待ち数：**
```
0(己) = 0  ← すぐ読める
1(所) = 2  ← 不と欲を待つ
2(不) = 1  ← 欲を待つ
3(欲) = 0  ← すぐ読める
```

**処理の流れ：**
1. 己を読む（待ち数0）
2. 欲を読む（待ち数0）→ 不の待ち数が0に
3. 不を読む → 所の待ち数が1に
4. 所を読む（すべての待ちが終わった）

**結果：己 → 欲 → 不 → 所**
**意味：己の欲せざる所**

## よくある質問

### Q1: なぜ「矢印」で表すの？

**A:** 「AをBより先に読む」という関係を、目で見てわかりやすくするためです。
矢印は「AからBへ」という流れを表していて、料理のレシピの「〇〇してから△△する」と同じ考え方です。

### Q2: 「待ち数」って何のためにあるの？

**A:** 「今すぐ読めるかどうか」を判断するためです。
待ち数が0なら「他の文字を待たなくていい」= 今すぐ読める、ということです。
料理でいえば「材料が全部揃った」という状態です。

### Q3: なんで「0から始めて1ずつ減らす」の？

**A:** これが一番シンプルで間違えない方法だからです。
「できることから順番にやる」というルールを機械的に実行しているだけです。

### Q4: もっと複雑な訓点（上中下点とか）にも使えるの？

**A:** 使えます！矢印の作り方のルールを追加するだけで、同じアルゴリズムが使えます。
基本的な考え方は全く同じです。

---

**このアルゴリズムは、料理の手順を考えるのと同じです。「これをする前に、あれをしないと」という順番を、一つずつ確実に処理していくだけなんです。**
