# 読み順解決アルゴリズムの解説

## 概要

漢文の書き下し変換において、最も重要な処理は「**訓点に基づいて読む順番を決定すること**」です。

このプロジェクトでは、**依存関係グラフ + トポロジカルソート**というアルゴリズムを使って、この問題を解決しています。

## 問題: なぜ読み順を決めるのが難しいのか？

漢文は中国語の語順（主語→動詞→目的語）で書かれていますが、日本語では語順が異なります（主語→目的語→動詞）。

訓点（返り点）は、日本語の語順で読むための「読み順の指示」です。

### 例: 己所不欲勿施於人

```
元の文字列（訓点付き）:
己ノ 所[レ] 不[レ] 欲セ 勿[レ] 施ス[二] 於 人ニ[一]

期待される読み順:
己ノ → 欲セ → 不 → 所 → 人ニ → 於 → 施ス → 勿

書き下し文:
己の 欲せ ざる 所、 人に 施す 勿れ。
```

訓点には複雑なルールがあり、単純な左から右への読みではありません：
- **レ点**: 次の文字を先に読んでから戻る
- **一二点**: 起点（一）から終点（二）へジャンプして読む
- **複合訓点**: レ点と一二点が組み合わさることもある

## 解決策: 依存関係グラフ

### 基本的な考え方

「AをBより先に読まなければならない」という関係を**依存関係**として表現します。

```
依存関係の例:
- 「欲」→「不」（「欲」を先に読んでから「不」を読む）
- 「不」→「所」（「不」を先に読んでから「所」を読む）
```

これらの依存関係を**グラフ構造**で表現すると、読み順を機械的に決定できます。

### グラフとは？

グラフは「ノード（節点）」と「エッジ（辺）」から成る構造です：
- **ノード**: 各文字（己、所、不、欲、...）
- **エッジ**: 依存関係（矢印）

```
欲 → 不 → 所
```

この矢印は「欲を読んでから不を読む」「不を読んでから所を読む」という順序関係を表します。

## アルゴリズム詳細

### ステップ1: 依存関係グラフの構築

訓点のルールに基づいて、各文字間の依存関係を作ります。

#### レ点のルール

`A[レ] B` という構造では、**Bを先に読んでからAを読む**

```
例: 不[レ]覚

依存関係: 覚 → 不
（「覚」を先に読んでから「不」を読む）
```

実装上は、隣接リスト（adjacency list）で表現します：

```typescript
// i番目の文字にレ点がある場合
if (kunten === Kunten.RE && i + 1 < n) {
  adj[i + 1].add(i);  // 次の文字(i+1) → 現在の文字(i)
}
```

#### 一二点のルール

`X[二] Y Z[一]` という構造では、**起点（一）を先に読み、その後に終点（二）を読む**

```
例: 施ス[二]於人ニ[一]

読み順: 人ニ → 於 → 施ス
（起点「一」→ その間の文字 → 終点「二」）
```

実装上は、起点を見つけたら、それより前にある終点を探します：

```typescript
// 起点（一、上、甲など）の依存関係
if (kunten.isStartingPoint) {
  const endPoints = this.findMatchingEndPoints(startIndex, words, kunten.jumpStrategy);

  // 起点 → 最後の終点
  adj[startIndex].add(lastEndPoint);

  // 起点の次の文字から最初の終点の前まで → 最初の終点
  for (let j = startIndex + 1; j < endPoints[0]; j++) {
    adj[j].add(endPoints[0]);
  }
}
```

### ステップ2: トポロジカルソート

依存関係グラフができたら、**トポロジカルソート**（topological sort）で読み順を決定します。

#### トポロジカルソートとは？

有向グラフにおいて、すべてのエッジ A → B に対して「Aを出力した後にBを出力する」という順序を保つようにノードを並べることです。

簡単に言えば：**依存関係を守りながら、全ての文字を並べる**

#### Kahn's アルゴリズム

このプロジェクトでは、**Kahn's アルゴリズム**を使用しています。

##### 手順:

1. **入次数（in-degree）を計算**
   - 入次数 = そのノードに入ってくるエッジの数
   - 「何個の文字がこの文字より先に読まれるべきか」を表す

2. **入次数が0のノードをキューに追加**
   - 入次数0 = 他の文字に依存していない = 最初に読める

3. **キューから1つ取り出して結果リストに追加**

4. **取り出したノードから出ているエッジを削除**（入次数を減らす）

5. **新たに入次数が0になったノードをキューに追加**

6. **キューが空になるまで繰り返す**

##### 実装:

```typescript
private topologicalSort(words: Word[], adj: Set<number>[]): number[] {
  const n = words.length;

  // 1. 入次数を計算
  const inDegree = Array(n).fill(0);
  for (let u = 0; u < n; u++) {
    for (const v of adj[u]) {
      inDegree[v]++;
    }
  }

  // 2. 入次数が0のノードをキューに追加
  const queue: number[] = [];
  for (let i = 0; i < n; i++) {
    if (inDegree[i] === 0) {
      queue.push(i);
    }
  }
  queue.sort((a, b) => a - b); // 元の順序を保持

  // 3-6. トポロジカルソート実行
  const result: number[] = [];
  while (queue.length > 0) {
    const u = queue.shift()!;
    result.push(u);

    // uの後続ノードの入次数を減らす
    for (const v of adj[u]) {
      inDegree[v]--;
      if (inDegree[v] === 0) {
        queue.push(v);
        queue.sort((a, b) => a - b);
      }
    }
  }

  return result;
}
```

## 具体例: 己所不欲勿施於人

実際の例で、アルゴリズムの動きを見てみましょう。

### 入力

```
インデックス: 0    1       2      3      4      5      6     7
文字:        己ノ  所[レ]  不[レ] 欲セ   勿[レ]  施ス[二] 於   人ニ[一]
```

### ステップ1: 依存関係グラフの構築

訓点のルールに従って依存関係を作成：

```
レ点のルール適用:
- 所[レ]: 3(欲) → 1(所)
- 不[レ]: 2(不) → 1(所)  ← 連続するレ点のため、実際には 3(欲) → 2(不)
- 勿[レ]: 5(施) → 4(勿)

一二点のルール適用:
- 人[一]が起点、施[二]が終点
- 7(人) → 6(於)
- 6(於) → 5(施)
```

依存関係グラフ（隣接リスト）:
```
0(己): なし
1(所): なし
2(不): 1(所)
3(欲): 2(不)
4(勿): なし
5(施): 4(勿)
6(於): 5(施)
7(人): 6(於)
```

視覚的には：
```
0(己)

3(欲) → 2(不) → 1(所)

7(人) → 6(於) → 5(施) → 4(勿)
```

### ステップ2: トポロジカルソート

#### 初期状態:

```
入次数:
0(己): 0  ← 依存なし
1(所): 1  ← 2(不)に依存
2(不): 1  ← 3(欲)に依存
3(欲): 0  ← 依存なし
4(勿): 1  ← 5(施)に依存
5(施): 1  ← 6(於)に依存
6(於): 1  ← 7(人)に依存
7(人): 0  ← 依存なし

キュー: [0, 3, 7] （入次数0のノード）
結果: []
```

#### 処理1: 0(己)を取り出す

```
結果: [0]
キュー: [3, 7]
```

#### 処理2: 3(欲)を取り出す

```
3(欲) → 2(不) のエッジを削除
2(不)の入次数: 1 → 0  ← 入次数が0になった！

結果: [0, 3]
キュー: [7, 2]
```

#### 処理3: 2(不)を取り出す

```
2(不) → 1(所) のエッジを削除
1(所)の入次数: 1 → 0

結果: [0, 3, 2]
キュー: [7, 1]
```

#### 処理4: 1(所)を取り出す

```
結果: [0, 3, 2, 1]
キュー: [7]
```

#### 処理5: 7(人)を取り出す

```
7(人) → 6(於) のエッジを削除
6(於)の入次数: 1 → 0

結果: [0, 3, 2, 1, 7]
キュー: [6]
```

#### 処理6: 6(於)を取り出す

```
6(於) → 5(施) のエッジを削除
5(施)の入次数: 1 → 0

結果: [0, 3, 2, 1, 7, 6]
キュー: [5]
```

#### 処理7: 5(施)を取り出す

```
5(施) → 4(勿) のエッジを削除
4(勿)の入次数: 1 → 0

結果: [0, 3, 2, 1, 7, 6, 5]
キュー: [4]
```

#### 処理8: 4(勿)を取り出す

```
結果: [0, 3, 2, 1, 7, 6, 5, 4]
キュー: [] （空になったので終了）
```

### 最終結果

```
読み順: [0, 3, 2, 1, 7, 6, 5, 4]

文字順:
0: 己ノ
3: 欲セ
2: 不（→「ざる」）
1: 所
7: 人ニ
6: 於（→スキップまたは助字として処理）
5: 施ス
4: 勿（→「なかれ」）

書き下し文:
己の 欲せ ざる 所、 人に 施す 勿れ。
```

## なぜこのアルゴリズムが優れているのか？

### 1. **明確な関心の分離**

- **依存関係の構築**: 訓点のルールに集中
- **読み順の決定**: グラフアルゴリズムに任せる

それぞれの責任が明確なので、バグが少なく、テストしやすい。

### 2. **拡張性**

新しい訓点（上中下点、甲乙丙丁点など）を追加する場合：
- 依存関係の構築部分に新しいルールを追加するだけ
- トポロジカルソート部分は変更不要

### 3. **複雑なケースに対応**

訓点が複雑に入り組んだ場合でも、依存関係さえ正しく表現できれば、トポロジカルソートが自動的に正しい順序を導き出します。

### 4. **理論的に堅牢**

グラフ理論とトポロジカルソートは、コンピュータサイエンスで確立された理論です。
- 正しさが証明されている
- 時間計算量: O(V + E) （V=頂点数、E=辺数）
- 効率的に実行できる

## まとめ

**依存関係グラフ + トポロジカルソート**のアプローチは：

1. 訓点のルールを「依存関係」として表現
2. Kahn's アルゴリズムで依存関係を守りながら文字を並べる
3. 結果として正しい読み順が得られる

このアルゴリズムにより、複雑な漢文の返り点も、明確かつ効率的に処理できます。
